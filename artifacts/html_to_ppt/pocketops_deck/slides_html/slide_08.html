项目概述

PocketOps 为工业车辆维保人员提供一套可离线运行的手机端 AI 诊断工作台，用端侧模型和本地知识图谱把现场故障、图片视频、维修步骤和工单提交连接起来。

端侧 AI 本机 NPU GraphRAG 多模态识别 离线诊断 动态工单
项目背景

工业车辆维保现场：网络弱、资料散、经验强依赖

典型痛点

  • 弱网环境 — 仓库、厂区、地下区域公网不可用
  • 资料分散 — 手册、工单、备件信息散落各系统
  • 经验依赖 — 新手难判断排查顺序，老手经验难复用
  • 工单缺项 — 设备编号、参数、步骤经常缺失
  • 图片未结构化 — 现场照片难以进入诊断流程

目标用户

一线维修工程师 · 售后服务人员 · 设备主管 · 维保外包团队

切入点

把"现场诊断 + 多模态采集 + 知识图谱 + 工单闭环"放进手机端

应用场景

仓储、制造、物流园区中的叉车、AGV、搬运车等工业车辆维保

核心方案

端侧 AI + 本地知识图谱 + 动态工单闭环

离线端侧诊断

本机知识图谱 + 本机 HTTP 推理服务，不依赖公网完成主诊断流程

图谱增强生成

图谱锁定事实（设备/症状/根因/备件/步骤），大模型只补充推理依据

多模态现场输入

文字、零件图片、铭牌图片、现场视频、故障码与参数

工作流

文字/图片/视频输入
→
本地 GraphRAG 匹配
→
端侧模型补充推理
→
诊断卡片 + 排查清单
→
动态工单 + 离线提交

动态工单闭环

连续问答、图片分析、设备上下文、用户补充参数和已完成排查项被汇总进工单，支持质量检查和离线队列

局域网增强服务

电脑侧 demo server 提供资料同步、附件下载、工单归档，不参与主诊断依赖

为什么适合骁龙平台

  • 实时性 — 现场交互需低延迟，端侧推理减少网络往返
  • 弱网可用 — 仓储厂区网络不稳定，端侧保障主流程
  • 隐私安全 — 设备编号、故障码等敏感数据留在本机
  • 多模态移动性 — 手机直接拍照、录视频、生成工单
  • 计算效率 — NPU 加速适合多模态模型持续交互

关键工程方法：启动时加载知识图谱 → 检查本机推理服务 → 轮询 /v1/models → 每次诊断前 /clear 清理状态 → 流式文本 + 非流式多模态

主线演示步骤

  • 启动就绪 — 登录、知识库加载、NPU 服务就绪
  • 图谱增强诊断 — 输入"3号叉车举升缓慢，液压压力偏低"，展示诊断卡片
  • 图谱依据 — 知识图谱视图、相似工单、排查清单
  • 图文追问 — 上传零件图"这是什么零件？"、铭牌图"配置号是多少？"
  • 端侧问诊 — 短症状输入，补充参数，排查清单全选后"生成动态工单"
  • 工单闭环 — 质量检查、离线队列、局域网补交

补充演示功能

  • 知识库未命中 — "AGV 偶发急停"，展示端侧泛化诊断
  • NPU 证明 — 触发推理 + adb logcat 导出 HTP 日志
  • 离线提交 — 关闭服务，工单进入待提交队列
  • 模拟蓝牙采集 — 电池电压 9.2V、液压 0.8bar
  • 历史搜索 — 按图谱/图片/视频/工单筛选

兜底策略：模型加载慢则先播录屏；服务不可用则展示离线队列；时间不足则只走启动→诊断→追问→工单四步
