# PocketOps 最新版拍摄脚本

## 片名

PocketOps：离线现场诊断与动态工单助手

## 视频规格

- 时长：3 分钟以内
- 画幅：竖屏 9:16
- 主画面：手机屏幕录制
- 辅助画面：工程师在叉车旁操作手机
- 场景：工业车辆维保现场
- 核心表达：PocketOps 在手机端完成文字诊断、图片/视频追问、问诊排查、知识图谱补全、动态工单生成和离线提交。电脑服务只是可选的局域网工单/资料服务，不依赖公网。

## 最新演示重点

- 文字诊断优先：上传过图片后，明确的文字故障问题仍走文字诊断和知识库链路。
- 图片上下文追问：铭牌照片可以连续追问配置号、叉车编号；独立文字故障仍回到文字诊断和知识库链路。
- 端侧问诊排查：短症状先生成本机问诊问题，用户补参数后继续收敛判断。
- 图谱增强诊断：命中本地知识图谱后，设备、症状、原因、步骤、备件和相似工单由图谱确定，端侧模型只补充推理依据、风险和待采集信息。
- 知识库未命中：明确提示未命中，进入端侧泛化诊断，也能生成工单草稿。
- 动态工单：点击“生成工单”时，会汇总当前连续问答、铭牌图片分析、设备上下文、参数、故障码和知识库结果，而不是固定取某一条回答。
- 工单质量检查：提交前本机检查设备编号、故障现象、关键参数、故障码、维修步骤等缺项。
- 离线提交队列：电脑服务不可用时进入本机待提交队列，可补交也可手动清空。
- 蓝牙诊断采集：当前为模拟故障码和参数采集，用于无真实设备时展示端侧诊断链路。
- 历史搜索：支持搜索和按图谱、图片、视频、工单、端侧分类筛选。

## 拍摄前准备

1. 手机安装最新版 PocketOps。
2. 手机准备模型目录：`/sdcard/GenieModels/<model-dir>/config.json`。
3. 首次启动前授予模型目录文件访问权限。
4. 准备叉车现场背景、清晰铭牌照片（配置号可读）、液压油渍照片、仪表照片、5-10 秒现场视频。
5. 可选：启动 `demo/mock_pocketops_server.py`，用于演示登录、资料同步、工单提交和云端工单查询。
6. 若展示离线提交，先关闭 demo server 或断开电脑服务，让工单进入待提交队列。
7. 蓝牙部分统一口径为“模拟诊断采集”，不要说成已接入真实车载控制器。

## 完整旁白稿

这是 PocketOps，面向工业车辆维保现场的端侧 AI 诊断助手。它把知识图谱、端侧模型、图片视频分析和工单闭环放进手机，现场没有公网也能继续工作。

启动后，应用先加载本地知识库，再检查手机上的 NPU 推理服务。电脑服务只是可选的局域网资料和工单服务，诊断推理仍在手机端完成。

明确的文字故障会优先进入文字诊断链路。命中本地知识图谱后，设备、症状、原因、步骤、备件和相似工单都由图谱确定；端侧模型负责解释为什么这样排查，以及现场还要补哪些数据。

图片追问不只是静态看图，也可以围绕铭牌照片连续追问配置号和这是几号叉车。随后用户补充“这台叉车举升缓慢，请给出维修建议”时，系统会把刚识别出的设备上下文带回文字诊断和知识库链路。

## 手机操作台本

1. 打开 PocketOps。
2. 登录，等待知识库和本机推理服务就绪。
3. 点击顶部“更多”，确认“端侧深度诊断”开启。
4. 输入：`3号叉车举升缓慢，液压压力偏低，请给出维修建议`。
5. 展示“图谱增强 + 端侧推理”诊断卡片。
6. 点击“图谱依据”，展示图谱和相似工单。
7. 返回诊断卡片，展示并勾选一项离线排查清单。
8. 点击“更多” -> “新会话”。
9. 输入：`5号叉车无法启动`。
10. 展示端侧问诊排查问题。
11. 输入：`仪表亮，电压9.2V，启动无反应，暂无故障码`。
12. 展示端侧问诊判断。
13. 点击输入栏 `+`，导入叉车铭牌照片。
14. 展示图片诊断回答，镜头重点停留在铭牌文字或配置号。
15. 输入：`配置号是多少？`。
16. 展示“图片上下文追问”回答配置号。
17. 输入：`这是几号叉车？`。
18. 展示系统根据铭牌/配置号匹配到对应叉车编号。
19. 输入：`这台叉车举升缓慢，请给出维修建议`。
20. 展示系统回到文字诊断/知识库链路，并使用刚识别的设备上下文。
21. 点击“生成工单”。
22. 展示“动态工单 · 图谱补全”或“动态工单 · 端侧整理”。
23. 展示端侧工单质量检查。
24. 点击“更多” -> “新会话”。
25. 输入：`AGV搬运车偶发急停，屏幕无明确故障码，重启后短时间恢复`。
26. 展示“知识库未命中 · 端侧泛化”。
27. 点击“生成工单报告”，展示未命中场景工单草稿。
28. 在工单页点击“提交工单”，电脑服务不可用时展示待提交队列。
29. 点击“更多”，展示提交队列、云端工单、清空队列、蓝牙、电脑服务、新会话。
30. 点击“清空待提交队列”，展示确认弹窗。
31. 可选：恢复 demo server，点击“立即提交待办工单”。
32. 点击“查看云端工单”。
33. 点击“更多” -> “蓝牙诊断采集”。
34. 等待模拟采集完成，点击“确认并诊断”。
35. 点击历史按钮，搜索“举升”或“工单”，切换筛选标签。
36. 恢复一条历史记录。

## 拍摄注意事项

- “蓝牙诊断采集”必须表述为模拟数据，不说成真实车辆控制器已接入。
- “云端工单”指 demo server 或电脑侧服务，不说成正式企业 CMMS。
- 若强调无公网，电脑服务统一说“局域网服务”。
- 文字诊断要拍清楚来源行：`图谱增强 + 端侧推理`。
- 铭牌照片要拍清楚配置号或设备编号文字，避免观众看不出图片识别依据。
- 配置号/几号叉车追问要拍清楚来源行：`图片上下文追问`，并说明它不会输出诊断报告卡片。
- 文字衔接要拍清楚：输入 `这台叉车举升缓慢，请给出维修建议` 后应回到文字诊断/知识库链路，并使用刚识别的设备上下文。
- 动态工单要拍清楚：点击生成工单前已有铭牌识别、配置号追问、叉车编号匹配和举升缓慢诊断，工单里应体现多轮上下文汇总。
- 未命中诊断要拍清楚来源行：`知识库未命中 · 端侧泛化`。
- 工单质量检查要拍清楚缺项提示，体现它不是静态页面。
- 顶部功能入口按最新版 UI：历史按钮 + 更多菜单，不再说“顶部蓝牙按钮”。
